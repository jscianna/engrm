# Security Degradation in Iterative AI Code Generation -- A Systematic Analysis of the Paradox

## Cite_Key
shukla2025security

## Problem
Security vulnerabilities evolve through iterative LLM feedback in AI-generated code.

## Method
Analyzed security degradation in AI-generated code through a controlled experiment with 400 code samples across 40 rounds of 'improvements' using four distinct prompting strategies.

## Data
400 code samples, 40 rounds of iterative improvements, four prompting strategies.

## Metrics
37.6% increase in critical vulnerabilities after five iterations.

## Findings
Iterative LLM refinement can increase security vulnerabilities, highlighting the need for human expertise in the loop.

## Limitations
Not specified.
